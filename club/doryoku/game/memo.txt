

■ 全体構成
----------------------------------------------------------------
21の関数・クラスを8つのサブシステムに分類。
トップレベルのコードは起動時に一度だけ実行される。

  [1] エンジン初期化     Three.js セットアップ・テクスチャ生成
  [2] ワールド生成       チャンク方式のボクセルワールド
  [3] 物理・衝突         AABB・重力・レイキャスト
  [4] プレイヤー操作     移動・ジャンプ・採掘・ブロック設置
  [5] インベントリ       ドロップアイテム・回収・スロット管理
  [6] 戦闘・弓矢         チャージ弓・矢の物理・ブロックダメージ
  [7] 描画・UI           フェイスカリング・ホットバー・HPバー
  [8] マルチプレイ       Socket.IO によるリアルタイム同期


----------------------------------------------------------------
■ ゲームループの流れ
----------------------------------------------------------------
requestAnimationFrame
  └─ gameLoop(now)
       ├─ deltaTime = (now - lastTime) / 1000   // ms → 秒
       ├─ updateChunks()   チャンク生成・解放・再構築
       ├─ update(dt)       ゲームロジック全体
       └─ render()         シーン描画


----------------------------------------------------------------
■ 関数・クラス一覧
----------------------------------------------------------------

エンジン初期化
  エンジン初期化（トップレベル）
    WebGLRenderer・Scene・Camera・ライト2灯をセットアップ。
    カメラ回転順をYXZにして視点操作を自然に。

  createTextureAtlas()
    256×256pxのCanvasに全ブロックテクスチャを手描き生成。
    4×4グリッド（64pxタイル）。外部画像ファイルは使わない。
    magFilter=Nearest でピクセルアートらしい見た目を保つ。

ワールド生成
  Chunk クラス
    16×32×16 = 8192バイトのUint8Arrayでブロックデータを管理。
    インデックス計算: x + z*16 + y*256。
    needsUpdateフラグが立つと次フレームでメッシュが再構築される。

  generateChunk(cx, cz)
    地形・壁・木・石構造物をプロシージャルに生成。
    Y=5が地面（草）、地下浅=土、深=石。
    ワールドは3×3部屋構成、内壁・外壁は白壁(id=7)。
    石の壁: チャンクあたり15%の確率でランダム配置。
    木: 6%の確率でスポーン（後述の密生防止あり）。

  buildChunkMesh(chunk)
    空気に隣接する面だけポリゴンを追加する「フェイスカリング」。
    全6方向を検査し、隣が空気または葉なら面を追加（2三角形=6頂点）。
    UV座標はブロック種類×面方向でアトラスタイルを算出。
    既存メッシュはgeometry.dispose()でGPUリソースを解放してから差し替え。

  updateChunks(playerX, playerZ)
    毎フレーム呼ばれ3フェーズで処理。
    ① 未生成チャンクをgenerateChunk()で生成
    ② プレイヤーから遠いチャンクをアンロード
    ③ needsUpdate=trueのチャンクをbuildChunkMesh()で再構築

  getBlock(x, y, z)
    ワールド座標→チャンク特定→ブロックID取得。
    負座標の剰余: ((x % 16) + 16) % 16 で正しく変換。
    多くの関数から呼ばれるコード中最頻出の関数。

  setBlock(x, y, z, val, sync)
    ブロックを変更し、破壊時はアイテムをドロップ、
    sync=trueのときSocket.IOで他プレイヤーに通知。
    チャンク境界上の変更は隣接チャンクのneedsUpdateも立てる。

物理・衝突
  checkCollision(px, py, pz)
    足元・腰・頭部の3高さ × 四隅4点 = 12点のAABBで判定。
    X/Y/Z軸を独立してチェックするため壁ずり移動が自然に発生。

  raycast(origin, dir, maxDist)
    0.05刻みのステップ法で最初のヒットブロックと面法線を返す。
    ブロック選択・壁ジャンプ検出・照準に使用。

プレイヤー操作
  update(deltaTime)
    移動・重力・ジャンプ・壁ジャンプ・採掘・弓チャージ・UI更新を担当。
    地上: 即時最高速度 / 空中: 緩やかな加速+空気抵抗の2段階。
    採掘時間: 木=1秒、石=2秒、葉=0.5秒。
    HPに応じてHPバーの色を緑→オレンジ→赤に変化。
    毎フレームサーバーにプレイヤー座標をemit。

  placeBlock(tx, ty, tz, normal, blockId)
    法線方向に1つずらした位置へブロックを設置。
    プレイヤーAABB（幅0.8×高さ1.5）との重複チェックあり。

  canPlaceBlock()
    ① 選択スロットがブロック種類か
    ② ブロック数が1以上か
    ③ カーソルが向いているブロックがあるか
    ④ PLACEABLE_BLOCKSリストに含まれるか
    の4条件をすべてチェック。

  executeBlockPlacement()
    canPlaceBlock()確認後にplaceBlock()を呼び、
    成功したらインベントリ数を減らしてUIを同期。

インベントリ
  spawnDroppedItem(x, y, z, itemId)
    1〜2個をランダムドロップ。水平飛び散り速度と上方向初速を付与。
    テクスチャはUVを手動書き換えでアトラスの該当タイルを使用。
    600フレームの寿命（約10分）で自動消滅。

  updateDroppedItems(deltaTime)
    重力（通常の3倍）・地面衝突・sin波浮遊アニメーション・
    自動回収（距離1.5以内）・寿命消滅を毎フレーム処理。

  collectItem(itemId)
    スロット2〜7に対してスタックまたは空スロットへ新規追加。
    スロット0=弓・スロット1=矢は固定。満杯時はfalseを返す。

戦闘・弓矢
  shootArrow()
    溜め量0: 速度10 m/s・ダメージ5
    溜め量1.0: 速度38 m/s・ダメージ30
    発射後に矢数と弓耐久値をそれぞれ1減らす。

  updateArrows(deltaTime)
    ① 敵との衝突判定→ダメージ
    ② 自/他プレイヤー衝突判定
    ③ 重力適用・座標更新
    ④ 先端経路を分割した精密ブロック衝突判定
    ⑤ 石ブロックへのダメージ（HP50%以下でひび石化）
    刺さった矢はブロックが壊れると落下開始。

描画・UI
  updateHeldItemMesh()
    選択スロットに合わせて手元の3Dモデルを切り替え。
    弓は3パーツ、矢は棒+円錐矢じりをGroupで合成。

  updateHotbarUI()
    ブロック・石・棒: CSSでアトラスを400%拡大して切り抜き表示。
    弓・矢: 絵文字で表示。数字は弓=耐久値・矢=残弾数・他=個数。

  render()
    矢・敵のメッシュをgameState配列の長さに合わせてプール方式で増減。
    ブロック選択枠はtargetBlock.hitがtrueのときのみ表示。

  gameLoop(now)
    isRunning=falseで即停止。deltaTime = (now - lastTime) / 1000。
    updateChunks → update → render → 次フレーム予約の順。

マルチプレイ
  addOtherPlayer(id, playerInfo)
    0.8×1.8×0.8の直方体にランダムカラーを付けてシーンに追加。
    Socket.IOイベント: currentPlayers / newPlayer / playerDisconnected /
    playerMoved / blockBroken / blockSet / arrowShot / takeDamage /
    enemyDamaged の9種類を処理。

----------------------------------------------------------------
■ gameState の主要なデータ保存
----------------------------------------------------------------
  player        位置(x,y,z)・視点角度(pitch,yaw)・HP・
                速度ベクトル・isGrounded・isDashing
  inventory     8スロット配列（0=弓・1=矢・2〜7=汎用）
  arrows        飛行中の矢のリスト
  enemies       敵エンティティのリスト
  blockHealth   Map("x,y,z" → HP) でブロックのHPを管理
  droppedItems  ワールドに落ちているアイテムのリスト
  keys          押されているキーの状態テーブル
  isCharging    弓チャージ中フラグ
  charge        弓の溜め量（0〜1.0）
  ammo          残り矢数
  bowDurability 弓の耐久値
